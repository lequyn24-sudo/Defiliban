## 1. Project Overview

### 1.1 Executive Summary

Defiliban is an AI-powered news platform specialising in **DeFi Protocols, Web3, and Crypto Finance**, designed as a **Typographic Broadsheet** — clean typography, editorial readability, and deep-dive specialist coverage. Articles are generated by **Claude AI** from RSS feeds, reviewed by an admin before publishing.

**Positioning:** "The go-to English-language source for DeFi protocol intelligence."

### 1.2 Business Objectives

| # | Objective | KPI | Target (Month 3) |
|---|-----------|-----|-----------------|
| 1 | Attract quality DeFi readers | Monthly Active Users | 30,000 MAU |
| 2 | Content velocity via AI | Published articles/day | 20–30 |
| 3 | Build subscriber base | Email subscribers | 8,000 |
| 4 | Monetize via sponsored content | Sponsored articles/month | 10 |

### 1.3 Target Users

**Primary — "The DeFi Researcher"**
- Age: 25–38, globally distributed (US, EU, SEA)
- Follows DeFi protocols closely: Uniswap, Aave, Lido, Curve
- Needs deep-dive analysis, not surface-level news
- Heavy desktop user, dark mode preferred
- English only

**Secondary — "The Protocol Builder"**
- Developer / founder building on DeFi protocols
- Needs technical accuracy: smart contract risk, oracle mechanics, bridge security
- Refers to articles for due diligence

**Non-users (out of scope MVP):**
- Retail traders needing price charts
- Vietnamese-language audience (v2)

### 1.4 Scope — MVP (v1.0)

**In scope:**
- [x] AI-generated articles (Claude API ← RSS feeds)
- [x] Admin review dashboard (approve / reject drafts)
- [x] DeFi-specialist category structure (Tier 1 + Tier 2)
- [x] Article reading — clean typographic broadsheet
- [x] Realtime price ticker (top 20 coins)
- [x] Newsletter email subscription
- [x] User accounts: bookmark, reading history
- [x] CMC data integration page
- [x] Sponsored articles + Press release sections
- [x] English only
- [x] Dark mode + Light mode

**Out of scope (v2+):**
- [ ] Vietnamese language
- [ ] Portfolio tracker
- [ ] Comments / community
- [ ] Paywall / paid subscription
- [ ] Native mobile app

```

Sub-nav:   Contextual — shows subcategories of active top-level section
           Always shows: Breaking (red) | Sponsored | Press Release
```

### 3.3 Navigation Guard Logic

```typescript
// middleware.ts
const PROTECTED = ['/account']
const AUTH_ONLY = ['/login', '/register']
const EDITOR_ONLY = ['/admin']

// Rules:
// 1. PROTECTED + no session → redirect /login?next=[path]
// 2. AUTH_ONLY + has session → redirect /account
// 3. EDITOR_ONLY + role not in ['editor','admin'] → redirect / with 403 toast
// 4. /api/pipeline/run without CRON_SECRET header → 401
```

## 5. Features & Functional Requirements

### 5.1 F-01: AI Content Pipeline

**User story (admin):**
> As an admin, I want the system to automatically fetch DeFi news and generate rewritten articles with expert analysis, so I have a steady stream of quality content to review.

**Acceptance criteria:**
- [ ] Vercel Cron triggers `/api/pipeline/run` every 2 hours
- [ ] Pipeline fetches last 24h articles from all RSS sources
- [ ] Deduplication: skip articles already processed (by source URL hash)
- [ ] Claude rewrites each article with DeFi specialist framing
- [ ] Correct category auto-detected from RSS feed + Claude classification
- [ ] Draft saved to Supabase with `status=draft`, `source_url`, `generated_at`
- [ ] Admin notified via email when queue > 10 pending drafts
- [ ] Pipeline logs stored in Supabase `pipeline_runs` table

**Edge cases:**
- Claude API timeout → retry once, then mark `status=failed`
- RSS feed unreachable → skip, log warning, continue other sources
- Article too short (< 200 words source) → skip
- Duplicate content from multiple RSS sources → deduplicate by source URL

---

### 5.2 F-02: Admin Review Dashboard

**User story:**
> As an editor, I want to review AI-generated drafts, make quick edits, and publish with one click.

**Acceptance criteria:**
- [ ] Queue page shows all `status=draft` articles, newest first
- [ ] Each queue item shows: title, category, source, word count, generated_at
- [ ] Review page shows full rendered article with inline edit capability
- [ ] Approve → `status=published`, triggers ISR revalidation
- [ ] Reject → `status=rejected` with optional reason
- [ ] Category can be overridden before publishing
- [ ] Title can be edited before publishing
- [ ] Bulk reject option for clearing low-quality batches

**Edge cases:**
- Two editors reviewing same draft simultaneously → last-write-wins with toast warning
- Publishing triggers ISR fail → retry once, alert admin

---

### 5.3 F-03: Article Reading

**User story:**
> As a DeFi researcher, I want to read deep-dive articles about specific protocols in a clean, distraction-free layout.

**Acceptance criteria:**
- [ ] Homepage shows 1 hero article + latest feed (right sidebar, numbered)
- [ ] Category pages paginate at 20 articles per page
- [ ] Sub-category filter pills work client-side (no page reload)
- [ ] Article loads in < 2.5s LCP
- [ ] Reading progress bar visible on scroll
- [ ] Estimated read time shown (words ÷ 200 wpm)
- [ ] Source attribution shown: "AI-generated · Based on [Source Name]"
- [ ] Related articles (same sub-category) shown below

---

### 5.4 F-04: Price Ticker

**Acceptance criteria:**
- [ ] Black ticker bar, full width, top of every page
- [ ] Shows top 20 coins: symbol, USD price, 24h % change
- [ ] Green (positive) / Red (negative) colour coding
- [ ] Refreshes every 30 seconds via polling
- [ ] Auto-scrolls on mobile
- [ ] Click coin → modal with 7-day sparkline + market cap + volume
- [ ] CoinGecko down → shows cached data with "Delayed" badge

---

### 5.5 F-05: Newsletter

**Acceptance criteria:**
- [ ] Email-only form (no name field)
- [ ] Double opt-in confirmation email via Resend
- [ ] Confirmation email sent < 60 seconds
- [ ] 1-click unsubscribe in every email
- [ ] Shown: inline footer, modal (after 60s reading), `/newsletter` page

---

### 5.6 F-06: User Account

**Acceptance criteria:**
- [ ] Register via email + password or Google OAuth
- [ ] Bookmark button on every article card and article page
- [ ] Optimistic UI update on bookmark toggle
- [ ] Reading history auto-tracked after 30s on page
- [ ] History retained 90 days, then auto-purged
- [ ] Settings: theme (light/dark/system)
- [ ] Account deletion deletes all data within 30 days (GDPR)

## 7. Security & Auth

> Full details in `SECURITY.md`

### 7.1 Auth Flow

```
Email/Password → Supabase Auth → JWT (1h) + refresh token (7d, httpOnly)
Google OAuth   → Supabase OAuth → same token flow
Admin routes   → check profiles.role in ['editor','admin']
Pipeline cron  → CRON_SECRET header validation (not user auth)
```

### 7.2 Role Matrix

| Action | Guest | User | Editor | Admin |
|--------|-------|------|--------|-------|
| Read articles | ✓ | ✓ | ✓ | ✓ |
| Bookmark | — | ✓ | ✓ | ✓ |
| Review queue | — | — | ✓ | ✓ |
| Approve/reject | — | — | ✓ | ✓ |
| Configure pipeline | — | — | — | ✓ |
| Manage users | — | — | — | ✓ |

### 7.3 Security Measures

- HTTPS enforced at Vercel edge
- CSP headers via `next.config.ts`
- Rate limiting: newsletter 3/IP/hour, login 10/IP/15min
- Zod validation on all API inputs
- Supabase RLS policies — users access only their own data
- No raw SQL — Supabase client only
- XSS — React escapes by default, AI content sanitised before storage

| Metric | Target | Strategy |
|--------|--------|----------|

- `robots.txt` — disallow `/admin`, `/api`, `/account`

- Canonical URLs on all pages

### 8.3 Accessibility

- WCAG 2.1 AA
- Semantic HTML throughout
- Alt text required on all article cover images (enforced in pipeline)
- Keyboard navigation full coverage
- Contrast ratio ≥ 4.5:1 body, ≥ 3:1 large text

### 8.4 Browser Support

Chrome, Firefox, Safari, Edge — last 2 versions. iOS Safari 15+. No IE.

### 8.5 AI Content Disclosure

Every AI-generated article must display:
```

Reviewed by the Defiliban editorial team before publication."
```
