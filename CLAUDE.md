## Project overview

Defiliban is a Next.js 14 news platform covering DeFi Protocols, Web3, and Crypto Finance.
Articles are AI-generated by Claude API from RSS feeds, reviewed by an admin, then published.
English only. Global audience. Dark-first design.

**Stack:** Next.js 14 (App Router) · TypeScript · Tailwind CSS v4 · Supabase · Claude API (`claude-sonnet-4-5`) · Vercel

## Design system — Dark Retro Digital

### Core palette

```css
--void:         #1A1A18;   /* Ticker bar, footer — deepest */
--charcoal:     #252525;   /* Page background — primary */
--surface:      #474744;   /* Cards, elevated surfaces */
--surface-2:    #323230;   /* Subnav, secondary surfaces */
--cream:        #E8E8C6;   /* Primary text, borders, accent */
--cream-dim:    rgba(232,232,198,0.55);  /* Secondary text */
--cream-faint:  rgba(232,232,198,0.12);  /* Subtle fills */
--cream-ghost:  rgba(232,232,198,0.06);  /* Hover states */

/* Functional — price data only */
--positive:     #4CAF7A;   /* Price up */
--negative:     #CF5C5C;   /* Price down, exploits */
--info-blue:    #7ABAEF;   /* Liquidity category */
```

### Rules — never break these

1. **Background is always `--charcoal` or darker.** Never use white or light backgrounds anywhere — not even in modals or cards.
2. **Text is always cream-family.** Never use black text (`#000`, `#111`, etc.) — invisible on dark bg.
3. **Accent = cream only.** No gold, no purple, no blue accent. The cream IS the accent.
4. **Borders = `1px solid rgba(232,232,198,0.18)`.** Use `0.45` opacity for emphasis (active states, strong dividers).
5. **No gradients, no glow, no blur.** Flat surfaces only. No `box-shadow` except focus rings.
6. **Positive/negative colors for financial data only.** Never decorate with them.

## AI pipeline

Vercel Cron: every 2 hours → `POST /api/pipeline/run`

Flow: RSS fetch → dedupe (source_hash) → Claude rewrites → `articles` table (`status=draft`) → admin reviews → approve → `status=published` + ISR revalidate

Claude model: `claude-sonnet-4-5`  
Max tokens: `1024` per article  
System prompt: authoritative DeFi analyst, no hype, English only, return JSON `{title, excerpt, body, category}`

Category must be one of the 17 slugs defined in SPEC.md §6.3. Claude classifies automatically.

---

## Database (Supabase)

Key tables: `articles` · `pipeline_runs` · `rss_sources` · `subscribers` · `profiles` · `bookmarks` · `reads`

Article status flow: `draft` → `published` | `rejected` | `failed`

RLS: enabled on all tables. Users access only their own rows. Pipeline uses service_role key.

See `DATAMODEL.md` for full SQL migrations.

## Auth (Supabase Auth)

- Email + password OR Google OAuth
- Access token: 1 hour JWT
- Refresh token: 7 days, httpOnly cookie
- Roles: `user` · `editor` · `admin` (stored in `profiles.role`)
- Middleware: `middleware.ts` guards protected + admin routes

## Code conventions

- **TypeScript strict mode** — no `any`, no `// @ts-ignore`
- **Server Components by default** — only add `'use client'` when component needs interactivity (onClick, useState, useEffect, polling)
- **No raw SQL** — always use Supabase JS client
- **Zod validation** on every API route input — reject before hitting DB
- **No hardcoded strings** for categories — import from `lib/constants/categories.ts`
- **ISR** for article pages — `export const revalidate = false` (on-demand via webhook)
- **Edge Runtime** for `/api/prices` — lowest latency for price data

## What NOT to do

- Never use white/light backgrounds — this is a dark-first design
- Never use `Playfair Display` or `C8A96E` gold — that was the old brand
- Never use `box-shadow` for card elevation — use background color difference instead
- Never add gradients or glow effects — flat surfaces only
- Never publish articles without admin approval — always `status=draft` first
- Never commit `.env.local` or expose `SUPABASE_SERVICE_ROLE_KEY` client-side
- Never use `any` type — use proper TypeScript or `unknown` with guards
- Never fetch Supabase from client when server component works — prefer RSC
